# 概要

## analysis_pythonとは
南が学生時代当時、C++で書くほど処理速度が要らなかったもの(特にプロットなど)をPythonにやらせていた時に、バックアップとして残していたgitです。  
基本的にメンテもしていなければ、修正もしていないので、当時の乱雑なまま書き残してあります。が、多分雑なりに、当時僕も初心者だったので、そんな高度な書き方はしておらず、初心者には読みやすいのではないかと思います。  
一方で当時の僕は、jupyter notebook/labというツールを知らないので、基本的に.pyを作ってはコマンドライン上でたたくというようなことをしていたので、それに対応した書きっぷりになっています。  
なので実験で解析しよう、ということであれば、適宜ライブラリ化して、jupyter notebook上でimportして使うのが賢いと思ったりします。

# 使うための環境設定
## versinとライブラリ
当時適当に作っていましたし、環境設定も適当にしていたのでコードを見ながら適当に書いておきます。
動かなかったら連絡してください。書き換えます。  
python version: 3.12以上(当時は3.11か3.12で使っていたと思います). 
必要なexternalライブラリ: numpy, matplotlib, pandas, scipyあたりがあれば大体動きます. 
あとは、画像処理をする場合はopencvやPIL、yamlファイルを使う場合はyamlをインストールしてください

## 開発環境管理の進め-uvマジおすすめ
これを機にuvとやらをお勧めしておきます。  
F研ではpythonはあまり流行っていますし、通常使いではpythonの環境を分けるというのも意識しないと思いますが、
社会人になってもpythonを使う、ないしソフトウェア系に行く、ちょっとでもかじる、という場合はプログラムの環境依存という概念に少しでも触れておいたほうが良いです。
実際僕も現職で、python 3.8の環境じゃないと測定機器を動かせないけど、解析はもっとモダンなpython 3.12でしたい...みたいなことが起こっています。
とは言いつつ、uvの仕方はwebの賢い方に任せます。  
https://zenn.dev/tabayashi/articles/52389e0d6c353a. 

個人的TIPSとしては、uvで`pip`を使うのはやめましょう。記事によってはpipを使うように言ってくるやつがいますが、uvを使っている効果が半減です。externalライブラリをインストールするときは`uv add ~~~`で追加しましょう。
その他は、習うより慣れろです。
